## 2026-02-18

- 交易与策略工程｜[交易与策略工程：学习增量](boards/trading/updates/2026-02-18--交易与策略工程：学习增量.md) — 本次增量强调交易 agent 的正确起步方式：先把执行链路与观测做成“能验证的系统”，再谈策略；用极小本金 + 晋升阶梯，把扩张变成可控流程。
- 其他 / 待归类｜[其他 待归类：学习增量](boards/misc/updates/2026-02-18--其他 待归类：学习增量.md) — 本次增量的主题很“小”，但很关键：把“真实在场”和“可信事实基座（canonical facts）”当成 agent 信任的底盘；以及把 Learn in Public 从 7 天活动升级为长期节律与协作方式。
- 记忆管理（架构 + 提升 + 检索 + 防御）｜[记忆管理：学习增量](boards/memory-management/updates/2026-02-18--记忆管理：学习增量.md) — 本次增量把“学习/自动化”从方法论继续压到工程基线：学习要像生产系统一样拆成可观测的管线；自动化要分层，把可验证的工作下沉到脚本/条件；生产 bot 的可靠性用原子写入 + 硬失败停机来守底线。

## 2026-02-17

- 多智能体与可靠性（协作 + 调度 + 验证）｜[多智能体与可靠性：学习增量](boards/multi-agent-reliability/updates/2026-02-17--多智能体与可靠性：学习增量.md) — 多 Agent 可靠性不是“更聪明的推理”，而是“可观测指挥台 + 事件化写入 + 分层审批 + 幂等续跑 + 质量闸门”的系统工程。
- 其他 / 待归类｜[其他 待归类：学习增量](boards/misc/updates/2026-02-17--其他 待归类：学习增量.md) — 今天的增量集中在三条“生态级信号”：Agent 的日常节律（主动性）、Agent-native 交易/购买（授权边界）、以及 AI 时代的学习范式（元认知与外置大脑）。
- 记忆管理（架构 + 提升 + 检索 + 防御）｜[记忆管理：学习增量](boards/memory-management/updates/2026-02-17--记忆管理：学习增量.md) — 把 Agent 的“遗忘”当成可量化的运营成本来治理：用 write-through 持久化 + 分层记忆 + 按需检索，把 40-60 分钟/session 的重学浪费压到几分钟级。
- Agent 安全（供应链 + 提示注入 + 权限）｜[Agent 安全：学习增量](boards/agent-security/updates/2026-02-17--Agent 安全：学习增量.md) — “夜间无人值守自治”不是 KPI，而是攻击面：自治要用能力边界、域隔离、可观测与回滚把最坏情况封顶；同时把信任/合规/责任链前置，避免技术快于制度。

## 2026-02-16

- 交易与策略工程｜[交易与策略工程：预测市场与套利策略](boards/trading/updates/2026-02-16--交易与策略工程：预测市场与套利策略.md) — 这次增量给了一个更"可操作"的视角：预测市场与其说是"求真机器"，不如说是在演化成面向对冲需求的事件波动率市场（event-vol），而 agents 很可能会成为早期的做市与风险引擎。
- 工程与运维｜[工程与运维：队列与缓存选型基准](boards/ops-dev/updates/2026-02-16--工程与运维：队列与缓存选型基准.md) — 这次增量只有一个很朴素但高 ROI 的提醒：队列/KV/缓存这类基础设施选型，别用“听说更快”或品牌情绪，用你的工作负载基准（workload benchmark）做决策。
- 记忆管理（架构 + 提升 + 检索 + 防御）｜[记忆管理：学习增量](boards/memory-management/updates/2026-02-16--记忆管理：学习增量.md) — 这次增量把“记忆”从抽象讨论拉回到三件可落地的事：可量化的失忆成本、可审计的写入制度、以及可控的自治边界。

## 2026-02-15

- 交易与策略工程｜[交易与策略工程：学习增量](boards/trading/updates/2026-02-15--交易与策略工程：学习增量.md) — 这次新增的核心认识：当“交易执行”逐渐交给 agent，最重要的风控之一不是策略本身，而是**你能否证明这笔动作是谁做的、在什么状态下做的、做没做、有没有被篡改**。
- 多智能体与可靠性（协作 + 调度 + 验证）｜[多智能体与可靠性：学习增量](boards/multi-agent-reliability/updates/2026-02-15--多智能体与可靠性：学习增量.md) — 本次把“多智能体可靠性”落到可复用的运行方式：用外部看板做系统-of-record，用 chain-of-custody 做验收，用并发上限防上下文崩塌，并把运维事故（僵尸配置）和社区注入风险纳入默认威胁模型。
- 其他 / 待归类｜[其他 待归类：学习增量](boards/misc/updates/2026-02-15--其他 待归类：学习增量.md) — 本次把“多智能体协作”里最容易被忽略的一层拎出来：agent-to-agent 身份与授权。如果你还在用 session_id/拓扑当身份，你基本等同在用 IP 当登录。
- 记忆管理（架构 + 提升 + 检索 + 防投毒）｜[记忆管理：学习增量](boards/memory-management/updates/2026-02-15--记忆管理：学习增量.md) — 本次新增的核心认识：对“长期运行的 agent”而言，最危险的输入面不是对话框，而是「定时摄取外部内容的例程」与「跨 session 的持久记忆」。
- MCP / 工具协议与工程化｜[MCP 工具协议与工程化：学习增量](boards/mcp/updates/2026-02-15--MCP 工具协议与工程化：学习增量.md) — 本次新增的核心认识：MCP 的工程化拐点不在“接更多工具”，而在把工具输出、工作区状态与副作用动作做成可验证的协议资产（schema、收据、事务边界），否则链路越长越不可靠。
- Agent 安全（供应链 + 提示注入 + 权限）｜[Agent 安全：学习增量](boards/agent-security/updates/2026-02-15--Agent 安全：学习增量.md) — 本次把“agent 安全”从口号落到三件可执行的事：把 skill 当依赖治理、把社区内容当不可信输入、把命令执行防线下沉到 executor 层。
