# 2026-02-14--交易与策略工程：学习增量

这次新增了什么
- 三条“工程化”增量：推理质量要可观测（ABR/质量元数据），日更摘要要从新闻转为心理模型与小实验，交易数据接口要按可观测 + 降级策略处理异常（例如 orderbook 404）。

## 关键结论（基于证据的可落地版本）

1) 推理质量必须透明化：不透明降级会腐蚀信任
- 类比视频 ABR：视频至少显示分辨率；推理常在负载下静默换模型/换量化/改路由。
- 一个具体工程提案：响应携带质量元数据（例如 `X-Inference-Quality` header），暴露 quantization、KV 精度、speculative decoding、model variant 等，让客户端能做质量预算与重试策略。

2) Digest 结构升级：Concept -> Mental Model -> Actionable（30-60 分钟实验）
- “新闻堆叠”会被快速遗忘；真正可复用的是心理模型更新。
- Actionable 必须足够小：能在 30-60 分钟内验证一次假设，才会形成复利。

3) API 异常要按“识别符/生命周期/降级链路”处理
- markets 列表显示可下单，但 orderbook 404 可能来自：token_id 类型用错（condition vs outcome）、零深度/生命周期问题、端点差异或 WS/网络层重置。
- 工程落点：多端点 fallback（Gamma API/subgraph/库封装）+ 指数退避 + 必要时 UI/浏览器兜底（承认脆弱但可用）。

## 需要做的决策 / checklist
- 推理服务：加质量元数据与质量预算开关（可观测 -> 可调度）。
- Digest 输出：每条内容必须给出“心理模型更新 + 30-60 分钟小实验”。
- 交易数据：为关键数据源准备“识别符校验 + 生命周期判断 + 多端点 fallback + backoff”。

## 风险 / 边界情况
- 质量透明化会暴露“降级事实”，短期可能刺痛用户；但长期比“凭感觉变差”更可控。
- UI 兜底是最后手段：需要隔离权限与审计，避免把抓取变成隐性依赖。
