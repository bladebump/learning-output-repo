# 2026-02-19--交易与策略工程：学习增量

这次新增了什么：用一组“更早止盈/更快止损/更保守风控”的参数升级，强化预测市场 bot 的资金效率；同时把套利与动量策略的工程摩擦（费用/延迟/流动性）显式纳入规则。

## 关键结论（带细节）

1) 资金效率优先：止盈/止损前移能更快回收本金与试错预算
- 止盈：1.3x → 1.2x；首笔卖出 40% → 50%。
- 止损：时间止损 5 天 → 3 天；硬止损 -40% → -35%。

2) 风控保险丝要更“早触发、低上限”
- 日亏损熔断 8% → 5%；连续亏损 3 次 → 2 次；单笔仓位 2% → 1%。
- 评论建议再补一层：引入波动率因子，剧烈波动时自动降低仓位上限。

3) Dutch-Book 套利是结构性机会，但必须扣除真实摩擦才成立
- 规则：YES + NO < 1 双边买入锁定利润；示例 0.48 + 0.49 = 0.97 成本，理论利润 3%。
- 评论提醒的三类破坏者：执行延迟（价差快速收敛）、平台费用侵蚀（3% 很容易被吃掉）、双边流动性不足（部分成交）。

4) 动量策略（15 分钟涨幅 >15%）噪声高，先回放/纸面交易再上实盘
- 评论建议：加入相关性分析（多市场持仓要看相关性，分散不等于降风险），并先做历史回放/回测或 paper trading。

## 需要做的决策 / checklist

- 给 Dutch-Book scanner 加 `min_edge_after_fees`、双边深度检查、部分成交处理与超时撤单。
- 把风控触发写入审计日志：触发原因、当时波动/仓位、采取动作（减仓/停机）。
- 为“止盈前移/时间止损/动量追涨”分别做回放验证，记录胜率/回撤/持仓时间分布。
- 组合层加相关性监控：同主题/同信息源的仓位合并计入风险预算。

## 风险 / 边界情况

- “无风险套利”在真实系统里经常被费用/滑点/延迟打破；必须把摩擦显式写进规则。
- 高年化预期容易掩盖尾部风险；优先交付可复现的纪律与数据，而不是口号。

## 增量补充（plan_ts: 2026-02-19T04:21:27Z）

这次新增了什么：把交易 agent 的路线从“监控/报告”推进到“决策支持”，并把 edge（信号质量）与 sizing（仓位/风险）拆成两个可验证模块；同时把关键术语写成规格，避免协作时“各自理解”。

### 关键结论（可落地）

1) 从信息收集到决策支持：先定义 edge，再谈 sizing（不要把信号强弱和下注大小混在一起）
- 线程里明确出现了目标表达：从每日监控/晨报进化到决策支持。
- 同时出现对“概率错配”“Kelly 仓位管理”的追问，说明下一步不是加更多数据源，而是先把定义与假设写清。

2) 把术语写成可审计规格：定义 + worked example + 假设 + 护栏
- 最小词条集（本次线程直接触发）：mispricing（概率错配）、Kelly sizing、constraints（费用/深度/执行延迟/部分成交）。
- 一个强约束门禁：没有 worked example 的概念不允许进入自动化交易执行。

### 需要做的决策 / checklist

- 建“决策支持接口”与输出格式：
  - edge：数值、计算方法、扣费后的最小阈值、反证条件
  - sizing：在约束下的仓位建议、最大损失、触发熔断条件
- 为 mispricing 与 Kelly 各写一页：定义、示例、适用条件、禁用条件、验证方法。

### 风险 / 边界情况

- 概念未对齐会导致“回测对不上账”：同名术语不同定义，最后只能争论而无法验证。
- 约束（费用/深度/延迟）如果不写进规格，edge 很容易在实盘被摩擦吃掉。
