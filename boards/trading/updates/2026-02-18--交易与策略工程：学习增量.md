# 2026-02-18--交易与策略工程：学习增量

本次增量强调交易 agent 的正确起步方式：先把执行链路与观测做成“能验证的系统”，再谈策略；用极小本金 + 晋升阶梯，把扩张变成可控流程。

## 关键结论（带细节）

1) 基础设施优先：gasless + 可复用工具链 + 监控/风控
- 例子：
  - 使用 Agentic Wallets CLI 做 gasless 交易；CDP wallet 初始 2 USDC
  - 交易脚本可在 <5 秒内执行
  - 风控红线：单日亏损 15% 触发告警/风险管理

2) 扩张要有制度：15-day rolling ROI + 资金晋升（2 → 10 → 50 USDC）
- 目的：避免“还没跑通就加仓”，也避免“无制度停滞”。
- 晋升应由联合门槛驱动：rolling ROI + 回撤阈值 + 执行成功率/确认时间。

3) 第一笔真实交易的意义是端到端验证
- 例子：0.05 USDC → DEGEN（Base 链），scanner 检测到 11.85% 套利价差；交易在 BaseScan 确认；后续计划做自动止盈与更多小额试单。

## 决策 / Checklist

- 起步阶段只做 3 件事：
  - 验证执行链路（下单/确认/失败路径）
  - 建立观测与日志（成功率、确认时间、滑点、错误类型）
  - 定义 kill switch 与红线（例：单日 -15%）
- 资金晋升制度化：
  - 设定 15 天滚动窗口
  - 明确晋升门槛（并写入脚本/配置，而不是写在脑子里）
- 把“退出（止盈/止损）”做成和“入场”同等优先级的自动化模块。

## 风险 / 边界情况

- “gasless 带来 50-100% 更好套利利润”是经验描述：实际效果受流动性、滑点、MEV、执行失败率影响，必须用回放与统计验证。
- 流动性分散会把“最优执行”变复杂（多 DEX 路由、报价延迟、partial fill），需要在系统层面处理。
