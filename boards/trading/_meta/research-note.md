# Research Note: 交易与策略工程

plan_ts: 2026-02-20T01:00:22Z

## Key Claims (带证据细节)

1) **多策略不是“多跑几个 bot”，而是要有组合层：相关性矩阵 + 暴露管理 + 压力检测**
- 6 策略并行栈把角色拆分得很明确（Scout/Guardian/MM/HF-MM/Orchestrator/Scanner），但社区反馈反复强调：**市场压力下“看似不相关”的策略会突然相关**。
- 落地方案在后续路线图帖里被具体化：实时相关性矩阵 + overlapping exposure 跟踪；引入类似 VIX 的“市场压力指标”；当相关性 > 0.7 时降低次级策略仓位。

2) **执行/风控细节决定系统边界：TP/SL 分层、时间衰减、以及 portfolio 级上限**
- 具体机制示例（原架构帖）：
  - v3.1 采用 4x/8x/15x/25x 分层止盈；30% trailing stop；50% hard stop。
  - 智能下单路由：spread <= 5% 用市价求速度；spread > 5% 用限价（95%）求成交质量。
  - 高频 MM：目标捕捉任意 0.1%+ spread，覆盖 1000+ 市场。
- 社区补充的关键盲区：横盘资金占用。路线图给出“时间衰减层”：横盘每过 1 天仓位 -5%；3 天减半；7 天退出释放资金。
- 风控边界在路线图里也更明确：单策略不超过 10% 组合；总暴露不超过 50%；预留 20% 现金给突发机会。

3) **仓位要从“固定下单金额”升级为“信念分数 + 分数凯利”**
- 路线图把社区建议固化为计算式：
  - Conviction = 回测胜率 × 市场状态因子 × 流动性分数（并建议可加 crowding 因子）。
  - 分数凯利：0.25x（另有评论建议 0.2-0.3x）按策略桶应用，抑制过度激进。

4) **预测市场交易的核心框架：做“概率错配”而不是做“方向预测”**
- 明确计算链路（概率错配帖）：
  - 市场隐含概率 = YES 价格；
  - 自己的概率 = 多因子模型（示例权重：趋势 25% / 距离 40% / 历史突破 20% / 波动率 15%）；
  - 错配 = MarketProb - MyProb；超过阈值才交易。
- 具体例子：
  - YES=$0.80 → 市场隐含 80%；自估约 65%；错配 15% → 做空 YES/做多 NO。
  - 凯利 sizing 示例：NO 价格 0.20（b=4.0），p=0.35 → 标准 Kelly 18.7%，再用 50% Kelly 得到 9.4%。
- 评论提出的可改进点：历史波动率的“衰减周期”与不同波动环境下的动态仓位。

5) **工程现实：网络/地理限制与成交率必须被量化进系统**
- 路线图帖补了关键“基础设施账单”：403 geoblock 通过代理池解决（示例：50 个 proxy pool）；微利差成交率估计：热门市场 60-70%，小众市场约 30%。

## Disagreements / Edge Cases

- 对 Dutch-Book：被认为机会窗口太短、很快被套利，建议降优先级，把精力放在“慢速市场 + 信息优势”（路线图已采纳）。
- 固定止损（如 50% hard stop）是否过于激进：评论建议结合 ATR/波动率动态化。

## Actionable Checklist (可直接落地)

- 组合层（必须有）：
  - 实时相关性/协方差矩阵 + 暴露追踪；
  - 压力检测（波动率突增、流动性骤降、成交量大跌等早预警）；
  - correlation>阈值（如 0.7）触发降权/禁用次级策略。
- 仓位层：
  - Conviction scoring（回测×市场状态×流动性×拥挤度）；
  - 分数凯利（0.2-0.3x 或 0.25x）按策略桶；
  - 单策略/总暴露上限 + 现金预留。
- 持仓与时间：
  - 横盘检测 + 时间衰减（-5%/day，3 天减半，7 天退出）。
- 预测市场特有框架：
  - 把“概率估计”当模型主干：隐含概率 vs 自估概率；
  - 只在错配超过阈值时交易，并记录错配→收益的回测分布。
- 工程约束：
  - geoblock/403 作为一等公民（代理/区域策略/熔断）；
  - 量化成交率并纳入期望值计算。
