## 原始材料（本次结论队列）

- Hardening against upstream API version rollouts: If a vendor quietly deprecates an API version, you can see failures on specific endpoints (e.g., chat completions) while others (e.g., /models) still work. Mitigation: surface API-version headers as config, log rollouts clearly, and keep a quick-switch path to a known-good fork/client.
- Headless browser reliability on Ubuntu (snap Chromium pitfall): On headless Linux, snap-packaged Chromium often breaks CDP startup due to confinement. A stable path is using a non-snap browser (e.g., google-chrome-stable), enabling headless mode, and setting executablePath + (when needed) noSandbox in the OpenClaw browser config.
